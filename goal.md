# goal.md — 目標契約

エージェントは「自分が done と思ったか」ではなく、**この基準が全部緑か**で完了を判断する。

## 完了の定義（数値で）

- `npm run lint` が clean
- `npm test` が全合格（CIゲート / 単体テスト）
- `npm run verify` が全合格（検査役ゲート / 受入基準）

## 受入基準（AC）

- **AC1**: 在庫が 1 以上 → `"in_stock"`
- **AC2**: 在庫が 0 → `"sold_out"`（売り切れ）
- **AC3**: 在庫がマイナス（返品処理中）→ `"sold_out"`

## 完了判定

- 検査役（別エージェント / `npm run verify`）が毎ターン AC を採点する。
- 作る役が "done" と宣言しても、AC が全部緑になるまで未完とする。

## 停止条件（4種）

| 種別 | 条件 | 動作 |
|------|------|------|
| 合格 | lint / test / verify がすべて緑 | PR を作成して人へ通知 |
| ロールバック | 変更で既存の緑テストが赤に転じた | 直前のコミットへ戻す |
| エスカレーション | 仕様が曖昧／AC を満たせないまま2周した | 人へ起票して引き継ぐ |
| タイムアウト | 10分 or 30ターン or $2 を超過 | 中断して現状を報告 |

## 権限ゲート（人の承認を残す）

- PR の作成まではエージェント。`main` への merge は人間が行う。
- 依存パッケージ追加・本番反映・ファイル削除は禁止（必要なら人へエスカレーション）。
